Encryption and Decryption:

If someone wants to send a secure message to the certificate holder, they use the holder’s public key to encrypt the message. Only the certificate holder can decrypt the message using their private key

The mutual authentication of client and server must be implemented at protocol level. It must be ensured that the identity of the server is checked by the client (authentication) before the client transmits credentials. Authentication must be carried out at application level or by the user.

Server Message Block (SMB)

Use of SMB 3.1.1: Introduced with Windows 10 and Windows Server 2016. It adds pre-authentication integrity checks and enhanced encryption capabilities Security Features

  • Authentication: SMB supports both user-level and share-level authentication. User-level authentication requires a valid username and password, whereas share-level authentication only requires a password for accessing shared resources.

  • Encryption: SMB 3.0 introduced encryption to protect data in transit, providing better security for sensitive information.

  • Access Control: SMB integrates with Windows access control mechanisms, allowing administrators to set granular permissions on shared resources.

  • Signing: SMB supports message signing to ensure the integrity and authenticity of messages exchanged between clients and servers.
