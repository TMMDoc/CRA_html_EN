Changing lost keys for the encryption and Backup Encrypted data at rest must be removed unencrypted for backups and encrypted with a separate backup key. The backup key must be secured in accordance with the specifications in this policy. If the runtime key of encrypted data at rest is lost, the data is restored from a backup and encrypted with a newly assigned key.
