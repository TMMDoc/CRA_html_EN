Praxis has a User role system). The authorization check is also used to filter the workflow tasks available to the current user when command panel is prepared for the selected item. The control system uniquely identify and authenticate all human users with password

Site-Admin role has full access to manage all accounts. There are no Non-human user accounts

  • Optionally: To switch from user level to administrative access, a daily changing 8-digit PIN is requested, which can be requested from the TRUMPF Service Center like on the machines

Passwords and other credentials as a server-side component (e.g. in the database of an IDM) must be cryptographically secured, non-recoverable form.

Passwords and other credentials as a client-side part of an API call, a database connection string or another connection setup must be stored in encrypted and integrity-secured form. The password for this secure storage must be secured in the associated application against easy reading.

SR1.5 ensures the proper management of authenticators (such as passwords, cryptographic keys, and biometrics) to maintain system security. It includes password policies such as complexity, expiration, and storage security.

Relevant Requirement Enhancements (REs) under SR 1.5:

SR 1.5 RE 1 – Password-based authentication (enforcing strong password policies).

SR 1.5 RE 2 – Multi-factor authentication (adding an extra layer of security beyond passwords).
