Logging and monitoring of events within the industrial control system environment ensures the confidentiality, integrity, and availability of critical infrastructure. It applies to all ICS environments, including all hardware, software, networks, and personnel involved in the operation and maintenance of these systems.

Log Generation

  • All critical systems and components must generate logs for security-relevant events, including but not limited to:

  • Authentication attempts (successful and unsuccessful)

  • Access to critical assets

  • Changes to system configurations

  • Privilege escalations

  • System startups and shutdowns

  • Security policy changes

Log Contents

  • Logs must contain sufficient detail to identify the following:

  • Date and time of the event

  • Source and destination IP addresses

  • User ID or system process triggering the event

  • Description of the event or activity

Timestamp Synchronization

  • All logging devices must be synchronized to a reliable time source using Network Time Protocol (NTP) to ensure the accuracy and consistency of timestamps.

Log Storage

  • Logs must be stored in a secure manner to prevent unauthorized access, modification, or deletion.

  • Logs should be retained for a minimum period as determined by regulatory requirements or business needs, typically not less than one year.

Log Access

  • Access to logs must be restricted to authorized personnel only. Role-based access controls (RBAC) should be implemented to enforce this restriction.
