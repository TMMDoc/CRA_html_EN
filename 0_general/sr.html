Security Requriemtnents (SR)

In IEC 62443-3-3, "SR" stands for Security Requirement. These are fundamental security capabilities that an Industrial Automation and Control System (IACS) must implement to protect against cyber threats.

Understanding SR in IEC 62443-3-3 IEC 62443-3-3 is part of the IEC 62443 series, which provides a framework for securing industrial control systems. Specifically, this part focuses on system security requirements and security levels.

The SRs (Security Requirements) are divided into seven foundational requirements (FRs), which categorize them based on security objectives.

The Seven Foundational Requirements (FRs) & Their Security Requirements (SRs)

  1. FR 1 – Identification and Authentication Control (IAC)

    • SR 1.1: Human user identification and authentication

    • SR 1.2: Software process and device authentication

    • SR 1.3: Account management

    • SR 1.4: Identifier management

    • SR 1.5: Authenticator management

    • SR 1.6: Wireless access management

    • SR 1.7: Strength of authentication

    • SR 1.8: Public key infrastructure (PKI) certificates

  2. FR 2 – Use Control (UC)

    • SR 2.1: Authorization enforcement

    • SR 2.2: Wireless use control

    • SR 2.3: Account use enforcement

    • SR 2.4: Access enforcement

    • SR 2.5: Session lock

    • SR 2.6: Remote session termination

    • SR 2.7: Concurrent session control

    • SR 2.8: Privilege management

    • SR 2.9: Control system access to external systems

  3. FR 3 – System Integrity (SI)

    • SR 3.1: Communication integrity

    • SR 3.2: Malicious code protection

    • SR 3.3: Security functionality verification

    • SR 3.4: Software, firmware, and information integrity

    • SR 3.5: Input validation

    • SR 3.6: Deterministic output

    • SR 3.7: Error handling

    • SR 3.8: Session integrity

  4. FR 4 – Data Confidentiality (DC)

    • SR 4.1: Information confidentiality

    • SR 4.2: Use of cryptography

    • SR 4.3: Key management

  5. FR 5 – Restricted Data Flow (RDF)

    • SR 5.1: Network segmentation

    • SR 5.2: Security-enforcing communication boundary

  6. FR 6 – Timely Response to Events (TRE)

    • SR 6.1: Audit log generation

    • SR 6.2: Continuous monitoring

  7. FR 7 – Resource Availability (RA)

    • SR 7.1: Denial-of-service protection

    • SR 7.2: Resource management

    • SR 7.3: Control system backup

Security Levels (SL) and Enhancements (RE) Each SR can have Security Level (SL) ratings (SL 1 to SL 4) that define increasing levels of protection against different adversary capabilities. Additionally, some SRs have Requirement Enhancements (REs) that strengthen security beyond the basic requirement.

The Security Requirements (SRs) in IEC 62443-3-3 are critical for defining cybersecurity measures in industrial control systems. They provide a structured approach to securing networks, devices, users, and processes, helping organizations implement robust cybersecurity based on their security level needs.
